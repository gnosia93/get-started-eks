Graviton 워크숍에서 애플리케이션 호환성 검토는 참가자들이 가장 두려워하는 "내 코드가 ARM에서도 돌아갈까?"라는 의문을 해소해 주는 단계입니다.
실무적으로 가장 많이 쓰이는 3단계 검토 프로세스를 워크숍에 녹여보세요.

### 1. 언어별 런타임 호환성 확인 (가장 쉬운 단계) ###
대부분의 현대적 인터프리터 언어는 코드를 수정할 필요가 거의 없습니다. AWS Graviton Getting Started 가이드에 따르면 아래 언어들은 런타임 버전만 맞으면 즉시 동작합니다.
Java/JVM: Java 8 이상 (Java 11 이상 권장)
Python/Node.js/Ruby: 대부분 그대로 동작하지만, C로 작성된 네이티브 모듈(Native Extensions)이 있는지 확인 필요
Go: Go 1.11 이상에서 다시 빌드(Recompile)만 하면 됨

### 2. 의존성(Dependencies) 및 라이브러리 검사 ###
코드 자체보다 바이너리 형태의 라이브러리가 문제입니다.
방법: Porting Advisor for Graviton 툴을 소개하세요. 이 오픈소스 툴을 실행하면 소스 코드를 스캔하여 ARM 아키텍처에서 호환되지 않는 라이브러리나 인라인 어셈블리 코드를 찾아줍니다.
실습 가이드: 워크숍에서 간단한 소스 코드를 주고 이 툴을 돌려보게 하면 참가자들이 "어디를 고쳐야 할지" 감을 잡기 좋습니다.

### 3. 멀티 아키텍처 도커 이미지 빌드 (핵심 실습) ###
요즘은 대부분 컨테이너 환경이므로, Docker Buildx를 사용하는 것이 표준입니다.
